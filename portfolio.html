HYPER TEXT TRANSFER PROTOCOL
================================

Introduction:
The Hypertext Transfer Protocol (HTTP) is an application layer protocol of the internet protocol suite that is primarily used for distribution and collaboration.

It works as the foundation of data communication for the World Wide Web.
Tim Berners-Lee started the creation of HTTP at CERN in 1989, and in a straightforward document titled HTTP Protocol Version 0.9, he described how a client and a server behaved. A web browser may be the client whereas a process, named web server, running on a computer that may be hosting more than one website may be the server.
This report is organized in sections explaining the following topics of HTTP.
• Message Type
• Message Sequence
• Message Syntax
• Type of Connection
The design of HTTP was made to help improve or establish connections between clients and servers. To improve response time, websites that normally have high traffic use web cache servers. To lessen network traffic, web browsers cache previously used web resources and reuse them whenever possible. By relaying messages with external servers, HTTP proxy servers at private network boundaries can make it easier for clients without a globally routable address to communicate.
The Internet protocol suite's framework served as the foundation for the design of HTTP, an application layer protocol. Since Transmission Control Protocol (TCP) is widely used, its definition presupposes an underlying and dependable transport layer protocol. On the other hand, HTTP can be modified to use shaky protocols like the User Datagram Protocol (UDP), for instance in HTTPU and Simple Service Discovery Protocol (SSDP).
The term hypertext was first used by Ted Nelson in 1965 while working on the Xanadu Project, which was an inspiration of American engineer Vannevar Brush’s 1930s vision of the microfilm-based information retrieval and management system called “memex” system. The original HTTP was invented at CERN by Tim Berners-Lee and his team. First proposed in 1989, the “WorldWideWeb” project was the main reason behind the design of HTTP. The first web server then went live the next year. The protocol used was the GET method which requests a webpage from a server and the response they got back from the server was only as a HTML page.
Some advantages of HTTP are as follows:
1. Addressing: It uses advanced scheme of addressing. In order to be easily recognised on the World Wide Web, it assigns IP addresses with names that are easily recognisable.
2. Flexibility: When an application requires extra features, HTTP has the ability to download add-ons or plugins and display the necessary information. Acrobat Reader and Flash players are examples of these.
3. Security: Each file in HTTP is downloaded from a separate connection before being closed. As a result, only one element at a time can be transferred from one webpage to another. As a result, there is less chance of interception during transmission which decreases the risks of getting hacked or getting your data stolen
4. Latency: The handshaking process will only occur in HTTP once the connection has been made. As a result, there won't be a handshake after a request. The connection's latency is greatly decreased as a result.
5. Accessibility: All HTTP pages are stored in internet caches called “page caches” when a page is loaded for the first time. As a result, the content loads quickly when the page is accessed again.
Some disadvantages of HTTP are as follows:
1. Data Integrity: Since HTTP does not use any encryption techniques, it is possible for someone to change the content. Because of this, HTTP is regarded as an insecure method that can compromise data integrity.
2. Data Privacy: Another issue with HTTP connections is privacy. Any hacker who is successful in intercepting the request has access to the entire contents of the web page. Additionally, they have the ability to collect private information like the username and password.
3. Server Availability: Clients do not take action to end the connection even after HTTP receives all the data it requires. So, the server won’t be available during this time.
4. Administrative Overhead: Multiple connections must be established by an HTTP in order to transmit a web page. The connection experiences administrative overhead as a result.


5. IoT Device Support: HTTP consumes more system resources, which increases power usage. Since today's IoT devices include wireless sensor networks, using HTTP is inappropriate.


HTTP Messages:
Data is transferred between a server and a client using HTTP messages. There are two kinds of messages: requests, which the client sends to the server to cause an action, and responses, which the server provides in response.
HTTP-Requests:
In the form of a request message an HTTP client sends the following HTTP request to a server. It consists of a request-line. Then a zero or more header header (General|Request|Entity) fields followed by CRLF. Then an empty line followed by the indication of the end of the header fields. Then the last thing is a optional message body.
GET,HEAD,POST,PUT,DELEETE,CONNECT,OPTIONS,TRACE are all the methods of request supported in HTTP/1.1.
HTTP-Responses:
A server responds with an HTTP response message after receiving and analysing a request message. First is the status-line which consists of the protocol version, a numeric status code, and a textual phrase that goes with it. The elements of the message line are separated by space SP characters.
The following 3-digit integer define the different classes of responses.
1. 1xx: Information
2. 2xx: Success
3. 3xx: Redirection
4. 4xx: Client Error
5. 5xx: Server Error
HTTP messages are made from textual information encoded in ASCII, and spans over multiple lines. These textual HTTP messages are rarely crafted by web developers: software, web browser, proxy or Web server performs this action.
HTTP Message Sequence:
HTTP messages are formed by materials that are appropriate to its type.
When an HTTP Input node receives a request for input or when an HTTP Request node receives a response from a web service, the bit stream's headers and body are parsed and represented within the message tree.
When the HTTP Reply node sends a response to the client or when the HTTP Request node sends a message, parsers construct a bit stream from the relevant portions of the message tree.

Message Syntax:
The message syntax for request status-line is as follows:
Request-Line = Method SP Request-URI SP HTTP-Version CRLF
The message syntax for response status-line is as follows:
Status-Line = HTTP-Version SP Status-Code SP Reason-Phrase CRLF

HTTP connection types:

The two main connection types of HTTP are persistent and non-persistent. The non-persistent connection has connection type 1.0 whereas the persistent connection has connection type 1.1.
Persistent: A persistent connection requires just one RTT to establish the connection before allowing the transfer of any number of objects.
Non-persistent: The non-persistent connection requires 2RTT plus file transmission time in total. The first RTT (round-trip time) is needed to establish the connection between the client and server. To request and return the object, a second RTT is required. This example exemplifies the transmission of a single object.
Conclusion:
Remember that HTTP is a "stateless" protocol, which means that every command is executed independently of every other command. Each HTTP request opened and closed a TCP connection according to the original specification. Multiple HTTP requests may be sent over a persistent TCP connection in more recent versions of the HTTP protocol (HTTP 1.1 and above), which improves resource usage. Large HTTP requests are regarded as application layer attacks or layer 7 attacks in the context of DoS or DDoS attacks and can be used to launch an attack on a target device.

IMPORTANCE OF HTTP:
--------------------------------------

  • • HTTP is the underlying protocol used by the World Wide Web.
    • HTTP is a request-response protocol used by clients and servers to communicate.
    • HTTP is used to transmit hypertext documents, images, videos, and other web-based content.
    • HTTP is a reliable, efficient and secure way to send data between web servers and clients.
    • HTTP is a stateless protocol, meaning that each request is independent of any other requests.
    • HTTP is the most widely used protocol for communication between web servers and clients.
    • HTTP enables the web browser to access web pages on the Internet.
    • HTTP facilitates the transmission of large files over the web.
    • HTTP is used to authenticate users and control access to web pages and data.
    • HTTP enables web applications to interact with other web applications and databases.
    • HTTP provides an efficient way to transmit data between different parts of an application.
    • HTTP is used to encode data before transmission and decode data after receipt.
    • HTTP is used to compress web pages and reduce their size before transmission.
